SecureCrypt User Guide

Getting Started

Welcome to SecureCrypt, your enterprise-grade file security solution. Follow these steps to begin:

1

Create an Account

Register with your email and set up a strong master password. This gives you access to all SecureCrypt features.

2

Configure Your Preferences

Visit the Settings section to customize your security preferences, default algorithms, and notification settings.

3

Upload Your Files

Click on the "Encryption" tab and select files you want to secure. You can drag and drop or browse your device.

4

Choose Security Method

Select between encryption (AES/DES) or hashing (SHA-256/SHA-512) based on your security needs.

5

Set Password

For encryption, create a strong password (minimum 12 characters recommended with mixed characters).

6

Process & Download

Click "Process File" and download your secured file. Encrypted files will have a .enc extension.

Encryption Guide

Always remember your encryption password. Without it, your files cannot be recovered. We don't store passwords.

Algorithm Selection

Algorithm Security Level Key Size Speed Recommended For
AES-256 Military Grade 256-bit Fast All sensitive documents, financial data
DES Basic (Legacy) 56-bit Very Fast Legacy systems only, non-critical data
SHA-256 Verification 256-bit hash Instant File integrity checks, digital signatures
SHA-512 High Verification 512-bit hash Fast Critical integrity verification, certificates

Encryption Process

When you encrypt a file with SecureCrypt:

  1. Your password is combined with a randomly generated salt
  2. We use PBKDF2 key derivation with 100,000 iterations to create a secure key
  3. The file is encrypted using AES in CBC mode with a random initialization vector
  4. Salt, timestamp, and IV are prepended to the encrypted data
  5. The resulting file can only be decrypted with the correct password

Best Practices

  • Use different passwords for different files
  • Store passwords in a secure password manager
  • Encrypt files before cloud storage for added security
  • Regularly update your encryption passwords
  • Use the password strength meter to ensure strong passwords

Cloud Storage Guide

All files stored in the cloud are automatically compressed and encrypted for maximum security.

Storage Features

Feature Description Benefit
Automatic Compression Files are zipped before storage Saves storage space and bandwidth
Secure Cloud Backup Encrypted storage on Dropbox servers Redundant, secure off-site storage
File Versioning Previous versions are maintained Recover from accidental changes
Access Anywhere Retrieve files from any device Work seamlessly across platforms

Storage Process

  1. Files are compressed into ZIP format to save space
  2. Metadata is preserved (filename, modification date)
  3. Files are securely transmitted to Dropbox servers
  4. You can access stored files from any device with your credentials
  5. Files can be retrieved individually or in bulk

Storage Limitations

  • Maximum file size: 2GB per file
  • Total storage: 50GB standard account
  • File retention: 30 days for free accounts
  • Supported formats: All file types

Best Practices

  • Encrypt sensitive files before storage
  • Organize files with clear naming conventions
  • Regularly clean up unused files
  • Use the bulk operations for managing multiple files

Security Guide

Security is a shared responsibility. Follow these guidelines to maximize your protection.

Security Features

Feature Description Protection Against
End-to-End Encryption Files encrypted before leaving your device Data interception, man-in-the-middle attacks
Password Hashing PBKDF2 with 100,000 iterations and salt Password cracking, rainbow table attacks
Secure Key Storage Encryption keys stored separately from data Data breach scenarios
File Expiration Automatic deletion after 5 minutes Unauthorized access to temporary files

Verification System

SecureCrypt includes a robust file verification system:

  1. Generate cryptographic hashes for your files
  2. Hashes are stored securely in separate location
  3. Verify file integrity at any time by comparing hashes
  4. Detect any tampering or corruption immediately

Threat Protection

  • Protection against brute force attacks
  • Automatic encryption of temporary files
  • Secure deletion of temporary files
  • Regular security updates and patches

Security Best Practices

  • Use strong, unique passwords for each file
  • Enable two-factor authentication where available
  • Regularly verify your stored files' integrity
  • Keep your software and systems updated
  • Be cautious of phishing attempts
  • Log out from shared devices after use

Compliance

  • GDPR - General Data Protection Regulation
  • HIPAA - Health Insurance Portability and Accountability Act
  • PCI DSS - Payment Card Industry Data Security Standard
  • SOX - Sarbanes-Oxley Act
